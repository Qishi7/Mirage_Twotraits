Four-State Gene Model

For gene \(i\), define the joint gene state

\[ S_i = (U_{i1}, U_{i2}) \in \{00, 10, 01, 11\}. \]

The four mixing proportions are

\[ \pi_{00} + \pi_{10} + \pi_{01} + \pi_{11} = 1. \]

Trait 1 initially represents autism and trait 2 schizophrenia. The model assumes independent studies with no sample overlap. Conditional on \(S_i\), the two trait-specific likelihoods are independent.

Trait-Specific MIRAGE Likelihood

For variant \(j\) in gene \(i\) and trait \(t\), MIRAGE conditions on total allele count \(T_{ijt}\). A non-risk variant follows

\[ X_{ijt} \mid T_{ijt}, Z_{ijt}=0 \sim \operatorname{Binomial} \left(T_{ijt}, \frac{N_{1t}}{N_{1t}+N_{0t}}\right). \]

For a risk variant with relative-risk multiplier \(R_{ijt}\),

\[ X_{ijt} \mid T_{ijt}, Z_{ijt}=1, R_{ijt}=r \sim \operatorname{Binomial} \left(T_{ijt}, \frac{rN_{1t}}{rN_{1t}+N_{0t}}\right). \]

Following the original MIRAGE model, the relative-risk multiplier is integrated over a user-specified Gamma prior. If \(g_{tc}(r)\) is the Gamma prior density for trait \(t\) and annotation category \(c\), then

\[ P(X_{ijt}\mid T_{ijt}, Z_{ijt}=1) = \int_0^\infty P(X_{ijt}\mid T_{ijt}, Z_{ijt}=1, R_{ijt}=r) g_{t,c_{ijt}}(r)\,dr. \]

The variant-level Bayes factor is therefore

\[ b_{ijt} = \frac{ P(X_{ijt}\mid T_{ijt}, Z_{ijt}=1) }{ P(X_{ijt}\mid T_{ijt}, Z_{ijt}=0) }. \]

The implementation supports trait-specific relative-risk hyperparameters. The first version does not force these parameters to be shared across traits.

Gene-Level Bayes Factors

For trait \(t\), let \(b_{ijt}\) be the variant-level Bayes factor and let \(\eta_{tc}\) be the trait-specific probability that a variant in annotation category \(c\) is a risk variant, conditional on the gene being active for trait \(t\). Then

\[ B_{it} = \prod_j \left[ (1-\eta_{t,c_{ijt}}) + \eta_{t,c_{ijt}} b_{ijt} \right]. \]

The implementation evaluates this in log space:

\[ \log B_{it} = \sum_j \log \left[ (1-\eta_{t,c_{ijt}}) + \eta_{t,c_{ijt}} b_{ijt} \right]. \]

Joint Likelihood

Ignoring factors that do not depend on model parameters, the gene-level contribution is

\[ D_i = \pi_{00} + \pi_{10}B_{i1} + \pi_{01}B_{i2} + \pi_{11}B_{i1}B_{i2}. \]

The observed-data log likelihood is

\[ \ell(\pi,\eta_1,\eta_2) = \sum_i \log D_i. \]

E Step

The posterior responsibility for each state is

\[ \tau_{i,00} = \frac{\pi_{00}}{D_i}, \quad \tau_{i,10} = \frac{\pi_{10}B_{i1}}{D_i}, \quad \tau_{i,01} = \frac{\pi_{01}B_{i2}}{D_i}, \quad \tau_{i,11} = \frac{\pi_{11}B_{i1}B_{i2}}{D_i}. \]

The implementation computes these with log-sum-exp normalization.

Marginal trait responsibilities are

\[ w_{i1} = \tau_{i,10} + \tau_{i,11}, \qquad w_{i2} = \tau_{i,01} + \tau_{i,11}. \]

The conditional posterior probability that a variant is a risk variant for its trait is

\[ \rho_{ijt} = \frac{\eta_{t,c_{ijt}} b_{ijt}} {(1-\eta_{t,c_{ijt}}) + \eta_{t,c_{ijt}} b_{ijt}}. \]

This remains trait-specific. The model does not introduce a four-state variant-level latent variable.

M Step

The four mixing proportions update as

\[ \pi_{ab}^{new} = \frac{1}{G}\sum_i \tau_{i,ab}. \]

Trait-specific annotation parameters update as

\[ \eta_{tc}^{new} = \frac{ \sum_i w_{it}\sum_{j:c_{ijt}=c}\rho_{ijt} }{ \sum_i w_{it}m_{itc} }, \]

where \(m_{itc}\) is the number of variants in gene \(i\), trait \(t\), category \(c\). Thus \(\eta_1\) and \(\eta_2\) are not shared.

Posterior Targets

The reported posterior probabilities are

\[ \operatorname{PP}_{trait1,i} = \tau_{i,10} + \tau_{i,11}, \]

\[ \operatorname{PP}_{trait2,i} = \tau_{i,01} + \tau_{i,11}, \]

\[ \operatorname{PP}_{pleiotropy,i} = \tau_{i,11}. \]

Bayesian FDR tables are computed separately for trait 1, trait 2, and pleiotropy by sorting posterior probabilities and averaging local posterior error probabilities.

Limitations

  • The current model assumes independent studies and no sample overlap.
  • Pleiotropy is gene-level, not variant-level.
  • The same gene can be pleiotropic even if different variants drive the two traits.
  • The current synthetic simulation helper is for software validation, not biological inference.
  • Weak data can make \(\pi_{11}\) or category-specific \(\eta\) values difficult to identify.
